## Goal

Build "DreamPlay Park," a single-page React+TypeScript application that presents an explorable fantasy amusement park where every attraction is a clickable classic mini-game (Snake, Brick Breaker, Tetris, 2048, Memory Match), with persistent local progress, achievements, and a dreamy animated aesthetic.

## Assumptions

- Node 20+ and npm are available in the container (use `npm`; no preference for pnpm/yarn).
- The implementer can use `sudo apt-get install` for any missing system packages.
- The container has outbound network access to the npm registry.
- No real audio assets are required; the optional ambient-sound toggle ships muted by default and uses generated WebAudio tones (no licensing risk, no asset pipeline).
- No accessibility audit beyond keyboard-navigable attraction buttons, focus-visible rings, and `prefers-reduced-motion` short-circuits for the park animations.
- The 5 Phase 1 games must each be a complete, playable experience. The 6 Phase 2 games from PRD §8 are NOT in this plan.
- 2048 is implemented with the standard 4×4 grid and Framer Motion `layout` animations on tiles; a single "Restart" button is the only in-game control (no undo).
- Memory Match uses emoji tiles (no external image assets).

### Key technical decisions

- **State management:** Zustand (not Redux) for the small persisted state. Pure achievement check functions in `src/store/achievements.ts` keep the logic testable.
- **Game loop:** a single shared `useGameLoop(deltaMs, isPaused)` hook so every Canvas game has a consistent pause/resume/cleanup pattern. `useRef` for the canvas; no `dangerouslySetInnerHTML`, no portal.
- **Routing:** React Router v6 with `createBrowserRouter`. Game param validated against `games/registry.ts`; unknown ids redirect to `/`.
- **Animations:** Framer Motion for camera zoom, hover, popup, and 2048 tile slides. CSS keyframes for the looping ambient background (clouds, balloons, fireflies) to keep the main thread free.
- **Sound:** WebAudio only. No asset files, no autoplay violations. Muted by default; first user click on the sound toggle unlocks the `AudioContext`.
- **Persistence:** one `localStorage` key `dreamplay.v1` with a versioned schema. Migration function in `gameStore.ts` so future schema changes are non-destructive.
- **Accessibility:** `prefers-reduced-motion` short-circuit at the Framer Motion level (`useReducedMotion()`), `aria-label` on all icon buttons, focus rings via Tailwind `focus-visible:ring-2`.
- **No backend, no auth:** PRD constraint. Score syncing is explicitly local.

### Risk controls

- Bundle size: dynamic-import `canvas-confetti` to avoid main-bundle bloat.
- Persistence corruption: `persist` middleware wrapped in a try/catch; on parse failure, fall back to initial state and log a warning.
- Audio autoplay blocks: gate WebAudio start on a user gesture.
- Framer Motion + `prefers-reduced-motion`: route the `useReducedMotion()` flag into every `motion.div` and skip the day/night cycle.
- Mobile touch input for Brick Breaker: bind `pointermove` on the canvas to paddle x; keyboard arrows are still supported on desktop.

This plan does not require any web access, browser scraping, or external downloads. All content is local, no third-party APIs. The PRD itself is the only reference doc.

## Out of Scope

- The 6 Phase 2 games from PRD §8: Flappy Bird, Space Shooter, Pac-Man Style Maze, Whack-A-Mole, Frog Crossing, Bubble Shooter.
- Multiplayer / online play (PRD §14 future expansion).
- Account system, cloud sync, leaderboards, social sharing.
- Real audio assets (the optional ambient sound uses generated WebAudio tones).
- Service-worker offline support / PWA installability.
- Internationalization beyond English strings.
- Server-side rendering or static export optimizations beyond the default Vite build.
- Automated end-to-end browser tests (Vitest + Testing Library is set up for unit tests of pure logic in `src/store/achievements.ts` and `src/utils/random.ts`; Playwright/Cypress is not included).
